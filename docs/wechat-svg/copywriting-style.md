# 公众号文案风格指南

这份文件规范的是**中文新媒体文案的质感**,不限制题材和创意。核心目标:让文字读起来像"人写的、有用的、带品味的",而不是"AI 批量产出的、堆词藻的、套路化的"。

## 一、最该避免的三种 AI 味

### 1. 三段式开头

**典型症状**:"在当今XX时代,XX已经成为XX。面对XX,我们该XX。本文将从XX三个角度,为您深入剖析XX。"

**问题**:信息密度为零,读者第一屏就走。

**替代写法**:
- 直接抛出核心价值:"5 分钟清理干净你的 C 盘,不用卸软件。"
- 用一个具体场景切入:"昨天同事的电脑蓝屏了,根本进不去系统。"
- 用反常识钩子:"打印机驱动不用装官网的。真的。"

### 2. 滥用排比和四字短语

**典型症状**:"高效、便捷、智能、专业"、"一键完成、轻松搞定、快速响应、稳定可靠"

**问题**:全是正确的废话,读者大脑会自动屏蔽。

**替代写法**:具体说清楚**快到什么程度、便捷在哪**。"处理 50GB 的临时文件夹,20 秒"比"高效快速"有说服力一万倍。

### 3. 感叹号轰炸

**典型症状**:"这真的太好用了!!" / "必看!!!" / "赶紧收藏!!!"

**问题**:显得廉价,且读者已经脱敏。

**替代写法**:用句号代替感叹号。情绪通过内容传达,不通过标点符号。**一篇文章感叹号不超过 3 个**是个好标准。

## 二、公众号文案的结构心智

### 标题(最重要)

- **具体 > 抽象**:「C 盘快满了?这 3 个文件夹敢删吗」> 「如何清理 C 盘」
- **数字 > 形容词**:「3 步」「5 分钟」「20GB」>「快速」「大量」
- **悬念 > 陈述**:「你以为删大文件就能腾空间?其实……」>「清理技巧分享」
- **不要用"干货"、"必看"、"收藏"等词**——降低点击率,且显得廉价

### 开篇钩子(前 100 字)

读者在公众号列表里只看得到前两行。这两行决定点不点进来。这两行必须:
- **回答"这跟我有什么关系"**:用读者的痛点或好奇点切入
- **不铺垫、不自我介绍**:别写"大家好,今天我们聊聊……"
- **有具体信息**:一个数据、一个场景、一个反常识的事实

### 正文组织

- **一个段落一个观点**:段落不超过 4 行(移动端阅读)
- **专业术语解释一次即可**:不要反复定义
- **小标题承担导航功能**:读者扫一眼小标题就知道在讲什么

### 结尾 CTA

- **明确单一动作**:下载 / 关注 / 转发 / 留言 / 阅读原文,只选一个
- **给理由**:"关注后下周会发 XX"比"欢迎关注"有力
- **不要自谦**:"文章可能有不足之处,欢迎批评指正"是 10 年前的写法

## 三、品牌声音的一致性

同一个公众号的所有文章应该**有一致的"人设"**。在开始写之前,先确认这个公众号的声音是:

- 严谨专家型(技术文档、深度分析)
- 亲切大哥型(工具推荐、使用教程)
- 机智段子手型(营销内容、品牌故事)
- 冷静观察者型(行业评论、数据报告)

不同定位的文字风格完全不同,混着用会让订阅者困惑。如果用户没说清楚,**参考其历史文章的语气**,或直接问用户"希望这篇的语气是哪种"。

## 四、排版节奏

### 图文比例

**移动端阅读,图片是呼吸节点**。长段文字之间必须穿插视觉元素,否则读者会划走。建议:

- 每 200-300 字至少一张图 / 一个 SVG 交互 / 一个卡片
- 图片/交互承担功能:说明、装饰、节奏、CTA
- 不要为了填空间而堆图——每个视觉元素要有存在的理由

### 长短句搭配

全是长句 → 显得啰嗦、AI 味浓。
全是短句 → 显得零碎、没有深度。

**混着来**。一段话里有 2 个长句分析,配 1 个短句做节奏强调。

**反例**:"本产品是一款专门针对 Windows 系统优化的工具软件,它具备多种实用功能。通过智能分析,它可以帮助用户高效清理系统垃圾文件。"

**改写**:"WMasterCleanup 只做一件事:清理 Windows 里你找不到的垃圾。20 秒扫完,告诉你哪些文件能删、哪些不能动。不装捆绑、不改注册表、不弹广告。"

### emoji 使用

- **工具类、科技类内容**:基本不用 emoji,显得不专业
- **生活类、情感类内容**:可以适度用,每段不超过 1-2 个
- **永远不要用这些**:📢 📣 🎉🎊 (会场型 emoji,显得廉价)
- **可以用的**:✅ → ※ 等功能性符号

## 五、产品推广类文案的特殊规则

### 不要堆功能清单

**反例**:"本产品具备以下功能:1. 清理 2. 优化 3. 修复 4. ……"

**正例**:选 1 个**读者最痛的场景**深入讲,其他功能一带而过。"C 盘满了就看这篇,打印机挂了就看那篇"——一篇文章解决一个问题。

### 不要自夸

"行业领先"、"业内顶尖"、"最专业"——这些词读者自动过滤。让**数据和案例**替你说话:"10 万 + 用户已安装"、"某知名企业采用"、"处理过 500GB 的真实案例"。

### CTA 要降低心理门槛

**反例**:"立即下载使用"
**正例**:"免费试用 7 天,不满意随时卸载"、"直接下载,无需注册"

把用户的疑虑(安全、成本、麻烦)前置回答。

## 六、中文排版细节

- 中英文之间**加空格**:"使用 Claude 分析" > "使用Claude分析"
- 数字和单位**加空格或不加看场景**:"20 秒"(自然) / "20GB"(通常不加)
- **引号用中文直角引号「」**或中文弯引号"",不用英文 ""(除非是代码)
- **破折号用 ——**(两个全角横线),不用 --
- **省略号用 ……**(两个全角点),不用 ...

## 七、自检清单

写完一段正文后,自问:

- [ ] 前两行有没有信息密度?读者会点进来吗?
- [ ] 段落是不是都在 4 行以内?
- [ ] 感叹号是不是不超过 3 个?
- [ ] 有没有"高效便捷智能专业"这种空话?
- [ ] CTA 是不是单一明确?
- [ ] 中英文空格、标点符号有没有统一?
- [ ] 读出声来,像不像一个真实的人在说话?

读出声这条最管用。AI 写的文案读起来会"顺得奇怪"——太工整、没有语气停顿。人写的文案会有起伏。
