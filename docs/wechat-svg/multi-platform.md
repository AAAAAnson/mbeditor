# 跨平台 SVG 适配指南

新媒体内容常常需要同时发布到多个平台。不同平台的 SVG 支持度差异巨大,**不能直接搬运**。这份文件列出主要平台的差异和适配策略。

## 三大平台对比

| 特性 | 微信公众号 | 微博头条文章 | 今日头条 |
|---|---|---|---|
| SVG 白名单限制 | 20 个动画属性 | **已全面开放**(仅禁 JS) | 严格限制,属性清单比微信更窄 |
| JavaScript | 禁用 | 禁用 | 禁用 |
| CSS 自由度 | 严格限制 | 较宽松 | 最严格 |
| `clip-path` / `mask` / `filter` | 全部禁用(2021.06 后) | 可用 | 禁用 |
| `position: absolute` | 禁用 | 可用 | 禁用 |
| 深色模式 | SVG 模块会失效 | 相对正常 | 不确定,建议规避 |
| 长文章字数限制 | 无硬性限制 | 有限制 | 有限制 |

## 微信公众号

主 SKILL.md 和其他 reference 文件主要面向微信。简要原则:

- 严守 20 个白名单属性
- 避开 HTML/CSS 禁用清单
- 所有效果用五大模式的组合来实现
- 深色模式下 SVG 会失效,要么接受降级,要么在文案层提示"浅色模式查看"

## 微博头条文章

**最友好的平台**。微博在 2023 年前后全面开放了 SVG 属性限制,只保留了一个硬性禁令:不能使用 JavaScript。这意味着:

### 可以用的(在微信上不能用的)
- 几乎所有 W3C 标准 SVG 属性的动画
- `clip-path`、`mask`、`filter` 等 CSS 视觉特效
- `position: absolute` / `relative` / `fixed`
- CSS 变量、CSS Grid、现代选择器
- `@keyframes` CSS 动画
- 复杂的 CSS 布局

### 依然不能用的
- `<script>` 标签、`onclick` 等事件属性、任何 JS 代码
- `<iframe>` 嵌入外部页面(部分允许,但有审核)

### 微博专属优化建议
- 充分利用 `filter: blur()` 等 CSS 特效做视觉层次
- 用 `@keyframes` 做更复杂的动画曲线(`cubic-bezier`)
- 可以做响应式的复杂布局

### 参考资料
微博开发者 JZCreativeStudio 发布的《微博SVG交互开发限制性规则》是最权威的参考。搜索该标题可以找到。

## 今日头条

**最严格的平台**。今日头条对 SVG 的支持明显比微信更窄,而且规则不公开。建议策略:

- **不为头条单独设计 SVG 交互**。性价比低。
- 准备纯静态的 SVG 图片(不带任何动画)即可
- 如果必须有动效,用 GIF 图片替代 SVG 动画
- 文章结构要比微信版更简单

## 跨平台发布的实操流程

如果一篇内容要发 2+ 平台:

### 1. 先做微信版本(最严格)

以微信为基线。因为微信的限制最多,能在微信跑起来的基本能在其他地方跑。

### 2. 按平台升级/降级

**发到微博**:可以把一些装饰效果升级。比如微信里用 `rect` 做的蒙版,微博版可以改用 `clip-path` 做更复杂的形状。

**发到头条**:降级到纯静态。把 SVG 动画换成 GIF,或干脆换成图片。

### 3. 单独做标题和开篇

不同平台的订阅逻辑和用户习惯不同:
- **微信公众号**:订阅关系为主,标题可以稍微含蓄、有品牌调性
- **微博**:算法推荐为主,标题要更直接、情绪化、带话题
- **今日头条**:下沉用户为主,标题要更口语、具体、有冲突感

### 4. 发布前各自测试

不要假设"微信能跑微博就能跑"。每个平台真机预览一次。

## 版本管理

如果频繁跨平台发布,建议:

- 给每篇文章建一个文件夹:`article-YYYYMMDD-slug/`
  - `wechat.html` - 微信版
  - `weibo.html` - 微博版
  - `toutiao.html` - 头条版
  - `source.md` - 原始文案
  - `assets/` - 共用素材

- 在 Git 里管理(哪怕是单人操作),方便回溯哪一版改了什么

## 一条原则

**为最严格的平台(微信)设计创意,为更宽松的平台(微博)做视觉升级**。反过来做(先做微博版再砍功能给微信)效率低且容易漏改。
